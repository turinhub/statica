请按以下步骤执行：

1. 先运行 `pnpm exec tsc --noEmit` 做类型检查，如有报错先修复。
2. 运行 `pnpm format` 格式化代码（仅提交与本次任务相关的改动）。
3. 运行 `pnpm build` 验证构建通过，如有报错先修复。
4. 使用 `git status --short` 与 `git diff --stat` 总结本次变更，至少包含：
   - 变更文件范围
   - 核心功能改动点
   - 验证结果（类型检查/构建）
5. 修改 `CHANGELOG.md` 文件，添加本次变更的记录；按需修改 `README.md` 文件，但注意保持精简。
6. 基于上述总结提交代码，不要 push 到 origin。
7. 最终汇报仅包含“变更总结 + 验证结果”，不要罗列第 1～5 步的执行过程。
